# 环境配置说明

## 一句话结论

> 本项目需要 **1 个外部 LLM API Key**（OpenAI 兼容格式，支持 DeepSeek / Moonshot / 智谱等国产替代）用于面试出题和 RAG 问答的文本生成；**Embedding 向量化使用本地 sentence-transformers 模型**，无需额外 API Key。未配置 LLM Key 时系统自动降级为规则引擎，所有功能仍可正常使用。

---

## 一、系统环境要求

| 依赖 | 最低版本 | 说明 |
|------|---------|------|
| Python | 3.9+ | 后端运行环境 |
| Node.js | 18+ | 前端构建 |
| pnpm | 8+ | 前端包管理（启动脚本自动安装） |
| Docker + Docker Compose | 最新版 | 用于运行 MySQL 和 Redis（也可用 brew 替代） |
| macOS / Linux | - | 启动脚本为 bash 编写 |

---

## 二、外部服务依赖（仅 2 个）

| 服务 | 用途 | 默认端口 | Docker 镜像 | 是否必需 |
|------|------|---------|------------|---------|
| **MySQL 8.4** | 业务数据持久化 | 3306 | `mysql:8.4` | 是 |
| **Redis 7** | 缓存层（岗位列表、统计数据） | 6379 | `redis:7-alpine` | 否（不可用时自动降级） |

两个服务均通过 `docker-compose.dev.yml` 自动拉起，无需手动安装。

---

## 三、端口占用

| 端口 | 服务 | 说明 |
|------|------|------|
| 3306 | MySQL | 数据库 |
| 6379 | Redis | 缓存 |
| 8000 | FastAPI (Uvicorn) | 后端 API + WebSocket |
| 5173 | Vite | 前端开发服务器 |

## 四、环境变量配置

### 后端环境变量

通过 `backend/.env` 文件配置（可选，均有默认值）：

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `DATABASE_URL` | `mysql+pymysql://root:root@127.0.0.1:3306/campus_recruit?charset=utf8mb4` | MySQL 连接地址 |
| `REDIS_URL` | `redis://127.0.0.1:6379/0` | Redis 连接地址 |
| `JWT_SECRET_KEY` | `change-this-in-production` | JWT 签名密钥 |
| `JWT_ALGORITHM` | `HS256` | JWT 算法 |
| `JWT_EXPIRE_MINUTES` | `1440`（24 小时） | Token 过期时间 |
| `CORS_ORIGINS` | `*` | 跨域允许的来源 |
| `APP_NAME` | `AI Campus Recruit Platform` | 应用名称 |
| `RAG_CHUNK_SIZE` | `400` | RAG 文本切片大小（字符） |
| `RAG_CHUNK_OVERLAP` | `80` | RAG 切片重叠字符数 |
| `RAG_TOP_K` | `5` | RAG 检索返回 Top-K |
| `EMBEDDING_MODEL` | `paraphrase-multilingual-MiniLM-L12-v2` | 本地 Embedding 模型名称 |
| `LLM_API_KEY` | （空，需手动配置） | 大模型 API Key |
| `LLM_BASE_URL` | `https://api.openai.com/v1` | 大模型 API 地址（支持 OpenAI 兼容格式） |
| `LLM_MODEL` | `gpt-3.5-turbo` | 使用的模型名称 |

### 前端环境变量

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `VITE_API_BASE` | `/api` | 后端 API 基础路径（通过 Vite 代理转发） |

> **重点：`LLM_API_KEY` 未配置时系统自动降级为规则引擎，不会报错。**

## 七、AI 功能技术实现（无外部依赖）

| AI 功能 | 技术实现 | 外部 LLM API | 本地模型 |
|---------|---------|-------------|---------|
| **岗位推荐** | 协同过滤 + 内容匹配 | 不需要 | 不需要 |
| **Embedding 向量化** | sentence-transformers 本地模型 | 不需要 | `paraphrase-multilingual-MiniLM-L12-v2` (~130MB) |
| **RAG 知识检索** | 混合检索（本地 Dense + TF-IDF Sparse） | 不需要 | 同上 |
| **RAG 回答生成** | LLM 基于检索上下文生成回答 | **需要** | - |
| **AI 面试出题** | LLM 生成面试题目 | **需要** | - |
| **企业 AI 筛选** | 算法评分（确定性） + LLM 评估文本 | **需要** | - |
| **学生模拟面试** | 算法评分 + LLM 生成反馈 | **需要** | - |
| **相似岗位推荐** | Jaccard 相似系数 | 不需要 | 不需要 |

> 所有标记"需要"的功能在 LLM API 不可用时**自动降级为规则引擎**，系统不会崩溃。

## 八、一键启动

```bash
# 克隆项目后，一条命令启动全部服务
pnpm dev
```

启动脚本 `start.sh` 自动完成：
1. 检测并安装 Python3 / Node.js / pnpm
2. 通过 Docker Compose 启动 MySQL + Redis
3. 创建数据库 `campus_recruit`
4. 创建 Python 虚拟环境并安装依赖
5. 安装前端 npm 依赖
6. 启动后端（8000）和前端（5173）

## 十、生产部署注意事项

以下配置在开发环境使用默认值即可，**生产部署时必须修改**：

| 配置项 | 开发默认值 | 生产建议 |
|--------|-----------|---------|
| `JWT_SECRET_KEY` | `change-this-in-production` | 更换为随机强密钥 |
| `LLM_API_KEY` | 空 | 配置为实际 API Key |
| MySQL 密码 | `root` | 更换为强密码 |
| `CORS_ORIGINS` | `*` | 限制为实际前端域名 |
| 测试账号密码 | `123456` | 删除种子数据或修改密码 |
| HTTPS | 未配置 | 生产环境应启用 HTTPS |

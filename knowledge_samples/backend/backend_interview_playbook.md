# 后端面试八股与工程实战手册

## 1. 计算机基础高频

### 1.1 HTTP 与 HTTPS

- HTTP 是明文传输，HTTPS = HTTP + TLS。
- TLS 解决三件事：加密、完整性、身份认证。

面试回答建议：

- 先讲握手，再讲证书校验，再讲性能开销和优化（会话复用）。

### 1.2 TCP 三次握手与四次挥手

- 三次握手保证双方收发能力可用。
- 四次挥手是全双工连接各自关闭的结果。

常见追问：

- 为什么不是两次握手？
- TIME_WAIT 为什么存在？

## 2. 数据库与缓存

### 2.1 MySQL 索引

- B+Tree 索引适合范围查询和排序。
- 最左前缀原则影响联合索引命中。
- 索引不是越多越好，写入和维护成本高。

排查慢 SQL 基本流程：

1. 用 `EXPLAIN` 看执行计划。
2. 确认是否走索引、回表次数、rows 估算。
3. 优化 SQL 或新增合适索引。
4. 回归压测并观察实际 RT。

### 2.2 Redis 高频场景

- 缓存穿透：查询不存在数据。  
  方案：布隆过滤器 + 空值缓存。
- 缓存击穿：热点 key 失效瞬间并发打 DB。  
  方案：互斥锁、逻辑过期、异步重建。
- 缓存雪崩：大量 key 同时过期。  
  方案：过期时间随机化、多级缓存、限流降级。

### 2.3 一致性问题

- 先更新数据库再删缓存（常见实践）。
- 使用消息队列做异步修复与重试。
- 对核心链路增加幂等和对账机制。

## 3. 并发与稳定性

### 3.1 限流

常见算法：

- 固定窗口计数器。
- 滑动窗口。
- 令牌桶。
- 漏桶。

生产建议：

- 按用户、IP、接口维度多级限流。
- 拒绝策略要可观测（日志 + 指标）。

### 3.2 幂等设计

场景：

- 下单、支付、消息消费重试。

方案：

- 幂等键（业务唯一 ID）。
- 数据库唯一索引 + 状态机。
- 消费端去重表。

### 3.3 事务与分布式事务

- 本地事务：ACID。
- 分布式事务常见方案：TCC、SAGA、消息最终一致性。

面试要点：

- 重点讲“失败补偿”和“可观测性”。

## 4. 服务治理与架构

### 4.1 微服务拆分

- 按业务边界拆分，而不是按技术层拆分。
- 避免过度拆分导致调用链过深。

### 4.2 服务发现与配置

- 注册中心：服务实例动态注册与健康检查。
- 配置中心：灰度发布与动态配置更新。

### 4.3 熔断降级

- 熔断：当错误率超阈值时快速失败。
- 降级：返回兜底数据，保障核心路径可用。

## 5. 后端面试题（含答题方向）

### 题目 1：如何设计一个高并发秒杀系统？

答题方向：

- 前置限流 + 验证码/风控。
- 预扣库存（Redis）+ 异步下单（MQ）。
- 幂等、去重、回滚补偿。
- 监控与扩容预案。

### 题目 2：如何保证接口幂等？

答题方向：

- 幂等键来源（客户端生成或服务端下发）。
- 唯一索引防重复写。
- 重试返回同一结果。

### 题目 3：慢查询怎么定位与治理？

答题方向：

- 指标告警触发 -> SQL 采样 -> 执行计划分析。
- 索引优化、SQL 重写、读写分离。
- 回归压测与上线灰度。

## 6. 项目复盘表达模板

推荐结构：

1. 业务目标（吞吐、RT、稳定性）。
2. 关键瓶颈（DB、缓存、线程池、网络）。
3. 方案设计（架构图、取舍、降级策略）。
4. 结果指标（RT 下降、QPS 提升、故障率下降）。
5. 遗留问题与后续计划。

可量化指标示例：

- P95 延迟从 420ms 降到 130ms。
- 峰值吞吐从 2k QPS 提升到 8k QPS。
- 故障恢复时间从 25 分钟降到 6 分钟。

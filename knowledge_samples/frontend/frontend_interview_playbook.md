# 前端面试八股与实战手册

## 1. 浏览器与渲染基础

### 1.1 从输入 URL 到页面展示

关键链路：

1. DNS 解析得到 IP。
2. TCP 三次握手建立连接（HTTPS 还会有 TLS 握手）。
3. 发送 HTTP 请求，服务端返回 HTML/CSS/JS。
4. 浏览器构建 DOM 树和 CSSOM 树，合成 Render Tree。
5. Layout（回流）计算几何信息，Paint（重绘）输出像素。
6. 合成线程把图层提交给 GPU 显示。

面试答法建议：

- 先讲主流程，再补性能关键点，比如：减少阻塞资源、降低关键渲染路径长度、按需加载。

### 1.2 回流与重绘

- 回流：影响布局，代价高。
- 重绘：不影响布局，只改视觉，代价相对低。

优化要点：

- 批量 DOM 修改，减少频繁读写布局属性。
- 动画优先使用 `transform/opacity`。
- 减少复杂选择器和过大 DOM 树。

## 2. JavaScript 核心高频题

### 2.1 原型链

- 每个对象有内部 `[[Prototype]]` 指向其原型对象。
- 函数有 `prototype` 属性，用于实例共享方法。
- 查找属性时沿原型链向上查找直到 `null`。

面试常问：

- `new` 做了什么？
- `__proto__` 和 `prototype` 区别？
- 如何实现继承（组合继承、寄生组合继承、class extends）？

### 2.2 事件循环与任务队列

- 宏任务：`setTimeout`、I/O、UI 渲染。
- 微任务：`Promise.then`、`queueMicrotask`。
- 每次宏任务结束后会清空微任务队列，再进入下一轮。

易错点：

- 不要把“同步代码属于宏任务”这种表达带进面试。
- 要说“同步代码在当前调用栈直接执行”。

### 2.3 防抖与节流

- 防抖：一段时间内只执行最后一次。
- 节流：固定时间窗口最多执行一次。

典型场景：

- 防抖：搜索输入建议。
- 节流：滚动监听、窗口 resize。

## 3. Vue3 / React 实战知识

### 3.1 Vue3 高频点

- 响应式核心：`Proxy + Reflect`。
- 组合式 API：`ref/reactive/computed/watch`。
- 组件通信：`props/emits/provide-inject/pinia`。
- 性能优化：异步组件、路由懒加载、虚拟列表。

### 3.2 React 高频点

- 渲染机制：函数组件每次 render 都会重新执行函数体。
- 状态更新：批处理 + 调度优先级。
- hooks 规则：只能在顶层调用，不在循环和条件中调用。
- 性能优化：`memo`、列表 `key` 稳定性、拆分大组件。

## 4. 工程化与性能优化

### 4.1 打包优化

- 路由级代码分割。
- 第三方依赖按需引入。
- 资源压缩（JS/CSS/图片）。
- 配置缓存策略（`cache-control` + 文件 hash）。

### 4.2 线上监控

- 错误监控：JS 运行时异常、资源加载失败、Promise reject。
- 性能监控：FCP/LCP/CLS/TTFB。
- 用户行为埋点：核心路径转化漏斗。

面试加分：

- 提到“监控 -> 定位 -> 修复 -> 回归”的闭环。

## 5. 前端面试题（含答题方向）

### 题目 1：虚拟 DOM 的价值是什么？

答题方向：

- 抽象跨平台渲染与声明式开发。
- 通过 diff 减少不必要真实 DOM 操作。
- 但不是绝对更快，仍需结合具体场景与优化策略。

### 题目 2：如何设计一个高可维护的前端项目结构？

答题方向：

- 分层：页面层、组件层、状态层、服务层。
- 统一错误处理、请求封装、类型约束。
- 规范化目录 + lint + commit 约束。

### 题目 3：首屏白屏优化怎么做？

答题方向：

- 关键资源内联、预加载、减少阻塞脚本。
- SSR 或预渲染提升首屏可见速度。
- 分析慢点：网络、打包体积、执行耗时。

## 6. 项目表达模板（STAR）

示例模板：

- S（背景）：业务目标和问题场景是什么。
- T（任务）：你负责的模块和约束条件是什么。
- A（行动）：技术选型、实现方案、关键取舍。
- R（结果）：量化收益（耗时降低、转化提升、错误率下降）。

推荐量化指标：

- 页面加载时间、JS 体积、错误率、接口成功率、用户转化率。

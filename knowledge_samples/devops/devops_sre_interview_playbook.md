# 运维 / SRE 面试八股与实战手册

## 1. Linux 与系统基础

### 1.1 高频命令与排障

- 进程与资源：`top`, `htop`, `ps`, `pidstat`。
- 网络定位：`ss`, `netstat`, `tcpdump`, `curl`。
- 文件系统：`df`, `du`, `lsof`。
- 日志定位：`journalctl`, `tail -f`, `grep`, `awk`。

面试表达建议：

- 不只是背命令，要讲“什么时候用、看什么指标、如何下结论”。

### 1.2 性能瓶颈排查框架

常见顺序：

1. 先看系统层：CPU、内存、IO、网络。
2. 再看应用层：线程池、连接池、GC、依赖超时。
3. 最后看外部依赖：DB、缓存、消息中间件。

## 2. Docker 与 Kubernetes

### 2.1 Docker 基础

- 镜像分层机制与缓存命中。
- 多阶段构建减小镜像体积。
- 容器资源隔离（namespace/cgroup）。

### 2.2 Kubernetes 核心对象

- `Deployment`：无状态应用滚动发布。
- `StatefulSet`：有状态服务稳定网络标识。
- `Service`：服务发现与负载分发。
- `Ingress`：七层流量入口。
- `ConfigMap/Secret`：配置与敏感信息管理。

### 2.3 常见故障题

题目：`Pod 一直 CrashLoopBackOff，如何排查？`

答题方向：

- 看 `kubectl describe pod` 事件。
- 看容器日志和启动命令。
- 检查配置挂载、环境变量、探针配置。
- 检查依赖服务连通性和权限。

## 3. 监控、告警与可观测性

### 3.1 三大支柱

- Metrics（指标）：延迟、吞吐、错误率、资源利用率。
- Logs（日志）：定位具体错误上下文。
- Traces（链路）：跨服务调用耗时与失败点。

### 3.2 SLI / SLO / SLA

- SLI：可量化指标。
- SLO：服务目标。
- SLA：对外承诺。

面试加分：

- 能把告警和 SLO 挂钩，避免“告警风暴”。

### 3.3 告警治理

- 聚合降噪，避免同类告警刷屏。
- 报警分级（P1/P2/P3）+ 响应机制。
- 故障后复盘（RCA）和行动项闭环。

## 4. CI/CD 与发布体系

### 4.1 标准流水线

1. 代码检查与单元测试。
2. 镜像构建与安全扫描。
3. 部署到测试环境并回归。
4. 灰度发布 + 监控观察。
5. 自动或人工放量。

### 4.2 发布策略

- 蓝绿发布：切换快，资源占用高。
- 金丝雀发布：低风险渐进放量。
- 滚动发布：平衡资源与稳定性。

### 4.3 回滚策略

- 保留可回滚镜像。
- 数据库变更遵循可回滚设计。
- 发布平台支持“一键回滚”。

## 5. 运维面试题（含答题方向）

### 题目 1：服务 RT 突然飙升怎么排查？

答题方向：

- 先确认是全局还是局部。
- 看资源与依赖健康。
- 看最近发布与配置变更。
- 给出止血方案（限流、降级、扩容）。

### 题目 2：如何设计一套报警体系？

答题方向：

- 指标分层（基础资源、业务指标、链路指标）。
- 阈值策略（静态 + 动态）。
- 值班流程（通知、升级、复盘）。

### 题目 3：如何提升故障恢复效率？

答题方向：

- 预案与演练（故障演习）。
- 自动化恢复脚本。
- 监控大盘和标准化排障手册。

## 6. 项目案例表达模板

示例：K8s 集群稳定性治理

- 背景：高峰期服务偶发超时，告警频繁。
- 行动：优化探针参数、调整 HPA、补齐依赖超时与重试策略。
- 结果：P95 延迟下降 45%，误报告警下降 60%，故障恢复时间缩短到 8 分钟。

推荐输出指标：

- 可用性（Availability）。
- 平均恢复时间（MTTR）。
- 变更失败率（Change Failure Rate）。
